webpack

Webpack은 JavaScript 애플리케이션을 위한 정적 모듈 번들러입니다.

웹 애플리케이션에서 사용되는 다양한 모듈, 자원, 그리고 의존성들을 하나로 묶어줍니다.

웹 애플리케이션은 여러 개의 JavaScript 파일, CSS 파일, 이미지 파일 등으로 구성됩니다. 이들 파일은 서로 의존 관계를 가지고 있을 수 있으며, 각각의 파일을 로드하고 관리하는 것은 복잡성을 증가시킬 수 있습니다. 또한, 모든 자원을 한 번에 로드하는 것은 성능에도 영향을 줄 수 있습니다. Webpack은 이러한 문제를 해결하기 위해 개발되었습니다. 애플리케이션의 모든 모듈과 의존성을 분석하여 하나의 번들 파일로 생성합니다. 이렇게 번들된 파일은 HTML 문서에 포함시키면 애플리케이션을 실행할 준비가 된 상태가 됩니다. 또한, Webpack은 로더(Loader)와 플러그인(Plugin)을 제공하여 다양한 기능을 확장할 수 있습니다. 로더를 사용하여 JavaScript가 아닌 파일들을 웹 애플리케이션에서 사용할 수 있는 형태로 변환하거나 처리할 수 있으며, 플러그인을 사용하여 번들된 파일을 최적화하거나 추가적인 작업을 수행할 수 있습니다.

Webpack개념

1. hello world 출력하기

import React from "react";
import ReactDOM from "react-dom/client";
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<h1>HELLO WORLD</h1>);
//hello world

2. JSX

JSX는 JavaScript XML의 약어로, React에서 UI를 구성하기 위한 문법입니다. JSX는 JavaScript의 확장 문법으로, XML과 유사한 문법을 사용하여 JavaScript 코드 안에 HTML 요소를 삽입할 수 있게 해줍니다.

const name = "webstoryboy";
const hello = <h1>hello {name}</h1>;
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(hello);
//hello webstoryboy

객체와 함수를 같이 사용한 경우

function helloName(){
    return name.nick;
}
const name = {
    nick : "webstoryboy",
}
const hello = <h1>Hello, {helloName()}</h1>;
